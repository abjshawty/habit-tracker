Backend API Endpoints

Method Path Description
GET /api/habits List all habits with 7-day done/total counts and today_done flag
POST /api/habits Create a new habit. Body: {"name": "Habit name"}
POST /api/toggles Toggle today's completion. Body: {"habit_id": 1, "completed": true}
GET /api/history Get 28-day completion log per habit
GET /docs/openapi.json OpenAPI 3.0 spec (JSON)

Response Models

// GET /api/habits response [ {"id": 1, "name": "Meditation", "done": 3, "total": 7, "today_done": true}, {"id": 2, "name": "Exercise", "done": 5, "total": 7, "today_done": false} ]

Database Schema

Tables

-- habits: Master list of tracked habits CREATE TABLE habits ( id SERIAL PRIMARY KEY, name TEXT UNIQUE NOT NULL ); -- logs: Daily completion records CREATE TABLE logs ( habit_id INT REFERENCES habits(id) ON DELETE CASCADE, date DATE NOT NULL DEFAULT CURRENT_DATE, completed BOOLEAN NOT NULL DEFAULT FALSE, PRIMARY KEY (habit_id, date) );

Default Seeds

Meditation, Exercise, Reading

Key Features

Optimistic Updates

UI updates instantly on tap, then syncs with server in background. Reverts on network failure.

Procedural SVG

All graphics drawn with code - no images. Seeded random ensures each run looks the same.

OpenAPI Docs

Built-in Swagger UI at /docs/ with full OpenAPI 3.0 spec

Auto Database

Backend auto-creates PostgreSQL database on first run - zero setup

Week Sparklines

Each habit shows 7-day progress mini-chart (last week)

Progress Arc

Circular arc around ring shows overall daily completion %
